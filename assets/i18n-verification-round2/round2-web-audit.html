Mappa Burp Suite

Cosa significa ogni funzione in stile Burp

Target

Costruisce una mappa del sito e dello scope. In OSINTPRO diventa un elenco chiaro di domini autorizzati ed evidenze pubbliche.

Proxy

Cattura il traffico del browser così un tester può capire richieste e risposte prima di modificare qualsiasi cosa.

Repeater

Lets a tester resend a request manually. OSINTPRO recommends this only for safe GET requests and evidence collection.

Decoder

Helps read encoded values such as URL encoding or base64. It is for understanding, not bypassing controls.

Comparer

Shows differences between two responses, such as before and after a security header fix.

Logger

Keeps a timeline of requests and responses. OSINTPRO connects that idea to report evidence.

Sequencer

Assesses token randomness. OSINTPRO explains the concept but does not collect sensitive tokens.

Scanner

Automates checks in Burp Suite. OSINTPRO does not run invasive scanning: it turns safe observations into owner-ready tasks.

Intruder

Automates payload attempts in Burp Suite. OSINTPRO does not automate guessing or abuse paths.

Collaborator

Detects out-of-band interactions. OSINTPRO only explains when this matters and leaves active testing to authorized labs.

Concetti di exploit

Classi di vulnerabilità spiegate in sicurezza

XSS

Rischio: High when it reaches authenticated users or sensitive data.

Revisione sicura per principianti: Review templates, escaping and CSP reports in an authorized environment.

Non automatizzato qui: No payload automation or stored attack attempts.

SQL injection

Rischio: High when it touches login, payment, admin or customer data.

Revisione sicura per principianti: Look for parameterized queries, ORM usage and database error handling.

Non automatizzato qui: No extraction, dumping or bypass attempts.

SSRF

Rischio: High when the app fetches user-provided URLs.

Revisione sicura per principianti: Inventory fetch endpoints, allowlists and cloud metadata protections.

Non automatizzato qui: No internal probing or metadata access.

Credential attacks

Rischio: Critical when login lacks rate limits or MFA.

Revisione sicura per principianti: Check rate limits, MFA coverage and leaked-secret handling.

Non automatizzato qui: Nessun password guessing, credential stuffing o furto di token.
