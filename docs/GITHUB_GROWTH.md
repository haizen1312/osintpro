# GitHub Growth Playbook

This is the free traffic plan for OSINTPRO.

## Repository Search

- Keep the README in English.
- Put high-intent keywords in natural text.
- Use GitHub topics:
  - `osint`
  - `passive-osint`
  - `domain-intelligence`
  - `web-audit`
  - `web-security`
  - `security-headers`
  - `brand-monitoring`
  - `wallet-osint`
  - `blockchain-analysis`
  - `fraud-investigation`
- Keep the live demo in the repository homepage field.
- Keep roadmap and issue templates active.

## Content To Publish

- Short build-in-public updates after every shipped feature.
- Sanitized screenshots of the Web Audit Lab, Entity Graph and Wallet Trace.
- Example reports for public domains.
- Small educational posts explaining CSP, HSTS, DMARC, wallet tracing and entity graphs.

## Community Targets

Share only with a passive/intelligence framing:

- indie builder communities
- web agency communities
- blue-team and threat intelligence communities
- OSINT learning groups
- crypto safety and scam-awareness communities
- GitHub project showcases

## What Not To Say

- Do not call it an exploit tool.
- Do not promise hacking results.
- Do not imply it can deanonymize wallets.
- Do not advertise brute force, bypasses or offensive payloads.

## Weekly Routine

1. Ship one visible improvement.
2. Update README or docs with the improvement.
3. Post a short build note.
4. Open or close a roadmap issue.
5. Share one sanitized workflow screenshot.
